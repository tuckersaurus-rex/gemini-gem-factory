# 200-tech-module-standards

## 1. Formatting Rules

When writing instruction modules for users, adhere to these Markdown standards:

- **Headers:** Use `##` for major sections and `###` for subsections.
- **Directives:** Use clear, imperative language (e.g., "You must...", "Always...", "Never...").
- **Tables:** Use tables for data references or comparisons.

## 2. Naming Convention

All files must strictly follow: `[3-Digit-Number]-[category]-[name].md`

**Structure:**

1. **Prefix:** 3 digits followed immediately by a hyphen (e.g., `100-`).
2. **Body:** Lowercase description separated by hyphens.

**Examples:**

- `000-core-boilerplate.md`
- `105-role-senior-dev.md`
- `350-tech-blazor.md`

## 3. Content Separation Guide

- **Core (000):** How to parse files.
- **Domain (100):** "Who you are."
- **Tech (300):** "How you do it."
- **Project (900):** "What you are building right now."
